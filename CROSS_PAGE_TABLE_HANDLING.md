# 跨页表格处理说明

## 问题背景

保险计划书的表格经常会跨多个页面，例如：
- 退保价值表可能有100行，分布在5-6页
- 每页可能有独立的 `<table>` 标签
- 有些PDF在每页重复表头

## 当前处理方案

### Prompt中的明确指示

在 `api/tasks.py:736` 的prompt中，已经明确要求：

```
2. 有些表格可能跨度好几个页面，但只算一张表，请完整识别
```

这意味着DeepSeek AI应该：
- ✅ 识别跨页表格为**一个表格**
- ✅ 统计所有页面的**总行数**
- ✅ 合并相同表格的字段信息

### DeepSeek的智能合并逻辑

DeepSeek会基于以下特征判断是否为同一表格：

1. **表格名称相同**：如果表格标题重复出现，识别为同一表格
2. **字段结构一致**：列名、字段结构相同
3. **数据连续性**：保单年度数字连续（1,2,3...或2,3,4...）
4. **上下文关联**：表格之间没有其他内容分隔

## 可能遇到的问题

### 问题1：跨页表格未被合并

**症状**：
- DeepSeek识别出多个名称相似的表格
- 例如：表格1和表格2都叫"退保價值表"，但被识别为2个表格

**原因**：
- OCR在分页时插入了干扰内容（页码、页眉、页脚）
- 表格标题略有差异（多了序号、页码等）
- 字段顺序不完全一致

**解决方法**：
1. 优化OCR质量，使用PaddleLayout API（已使用）
2. 手动修改 `tablesummary`，合并重复表格
3. 调整Prompt，增加合并规则示例

### 问题2：不同表格被误合并

**症状**：
- 两个不同的表格被识别为一个
- 例如："退保价值表"和"身故赔偿表"被合并

**原因**：
- 两个表格的字段结构非常相似
- 都包含"保单年度终结"列
- DeepSeek过度合并

**解决方法**：
1. 在Prompt中强调"表格名称不同则为不同表格"
2. 人工检查 `_table1.txt`, `_table2.txt` 等文件
3. 如有误合并，手动分拆

## 最佳实践

### 1. 信任DeepSeek的合并

在大多数情况下，DeepSeek能够正确识别跨页表格。Prompt中已明确指示，且DeepSeek支持大context。

### 2. 重点检查行数

如果某个表格应该跨5页，但行数只有20行：
- ❌ 可能未正确合并
- ✅ 应该有80-100行

### 3. 对比字段完整性

- 第1页：保单年度终结, 保证金额
- 第2页：保单年度终结, 保证金额, 非保证金额（新增列）
- → 合并后：保单年度终结, 保证金额, 非保证金额

## 技术细节

### OCR标记格式

PaddleLayout OCR返回的表格格式：

```html
<table>
  <tr>
    <td>保单年度终结</td>
    <td>保证金额</td>
  </tr>
  <tr>
    <td>1</td>
    <td>10000</td>
  </tr>
</table>

<!-- 分页符或其他内容 -->

<table>
  <tr>
    <td>保单年度终结</td>
    <td>保证金额</td>
  </tr>
  <tr>
    <td>21</td>
    <td>50000</td>
  </tr>
</table>
```

这两个 `<table>` 应该被识别为一个表格，行数=40（假设每个表格20行）。

### DeepSeek处理流程

1. **读取完整OCR内容**（现已无字符限制）
2. **识别所有包含"保单年度终结"的表格**
3. **分析表格间的关联性**：
   - 名称、字段、数据连续性
4. **合并相同表格**
5. **返回去重后的表格列表**
