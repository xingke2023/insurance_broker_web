# 客户案例管理后台使用指南

## 快速访问

### Admin 地址
```
http://localhost:8007/admin/api/customercase/
```

### 登录信息
- 使用管理员账号登录 Django Admin
- 在左侧菜单找到 "客户案例" 或直接访问上述URL

---

## 界面功能说明

### 1. 列表页面

#### 显示列

| 列名 | 说明 | 样式 |
|------|------|------|
| **案例标题** | 案例的标题 | 纯文本 |
| **人生阶段** | 扶幼保障期/收入成长期/责任高峰期/责任递减期/退休期 | 纯文本 |
| **客户年龄** | 客户年龄 | <span style="color: #3498db">蓝色加粗</span> |
| **年收入** | 格式化显示（千位分隔符） | <span style="color: #27ae60">绿色加粗</span> |
| **年缴保费总额** | 格式化显示（千位分隔符） | <span style="color: #e67e22">橙色加粗</span> |
| **推荐产品数** | 推荐产品数量 | <span style="color: #9b59b6">紫色加粗</span> |
| **是否启用** | ✔️ / ✖️ | 勾选框 |
| **排序序号** | 数字越小越靠前 | 数字 |
| **创建时间** | 创建日期时间 | 日期时间格式 |

#### 筛选器（右侧）

1. **人生阶段筛选**
   - 全部
   - 扶幼保障期
   - 收入成长期
   - 责任高峰期
   - 责任递减期
   - 退休期

2. **启用状态筛选**
   - 全部
   - 是（启用）
   - 否（禁用）

3. **创建时间筛选**
   - 今天
   - 过去7天
   - 本月
   - 今年

#### 搜索框（顶部）

**可搜索字段**:
- 案例标题
- 案例详细说明
- 家庭结构
- 保险需求

**使用方法**: 在搜索框输入关键词，按回车

#### 批量操作（底部下拉菜单）

1. **✅ 启用选中的案例**
   - 选中一个或多个案例
   - 选择此操作
   - 点击"执行"

2. **❌ 禁用选中的案例**
   - 选中一个或多个案例
   - 选择此操作
   - 点击"执行"

## 2. 添加/编辑案例

### 表单分组

#### 📋 基本信息

| 字段 | 类型 | 说明 | 必填 |
|------|------|------|------|
| **案例标题** | 文本框 | 例如："35岁单身专业人士保险规划" | ✅ |
| **人生阶段** | 下拉选择 | 5个选项 | ✅ |
| **是否启用** | 勾选框 | 默认勾选 | ✅ |
| **排序序号** | 数字框 | 默认0，数字越小越靠前 | ✅ |

#### 👤 客户资料

| 字段 | 类型 | 说明 | 必填 |
|------|------|------|------|
| **客户年龄** | 数字框 | 例如：35 | ✅ |
| **年收入** | 数字框 | 例如：800000 | ✅ |
| **家庭结构** | 文本框 | 例如："已婚，2个子女（5岁、8岁）" | ✅ |

#### 💼 保险需求

| 字段 | 类型 | 说明 | 必填 |
|------|------|------|------|
| **保险需求** | 多行文本框 | 描述客户的保险需求 | ✅ |
| **预算建议** | 文本框 | 例如："年收入的10-15%" | ❌ |

#### 🎯 推荐产品

| 字段 | 类型 | 说明 | 必填 |
|------|------|------|------|
| **推荐产品列表** | JSON文本框 | 见下方格式说明 | ✅ |
| **年缴保费总额** | 数字框 | 所有推荐产品的年缴保费总和 | ✅ |

**推荐产品JSON格式**:

```json
[
  {
    "product_name": "储蓄计划A",
    "company": "友邦保险",
    "annual_premium": 50000,
    "coverage_type": "储蓄",
    "reason": "稳健增值，长期收益可观"
  },
  {
    "product_name": "重疾保险B",
    "company": "保诚保险",
    "annual_premium": 30000,
    "coverage_type": "重疾",
    "reason": "全面覆盖100种重疾"
  },
  {
    "product_name": "医疗保险C",
    "company": "宏利保险",
    "annual_premium": 20000,
    "coverage_type": "医疗",
    "reason": "无限额住院保障"
  }
]
```

**字段说明**:
- `product_name` - 产品名称（字符串）
- `company` - 保险公司（字符串）
- `annual_premium` - 年缴保费（数字）
- `coverage_type` - 保障类型（字符串：储蓄/重疾/医疗/寿险/意外等）
- `reason` - 推荐理由（字符串）

#### 📝 案例详情

| 字段 | 类型 | 说明 | 必填 |
|------|------|------|------|
| **案例详细说明** | 多行文本框 | 案例的详细描述和分析 | ✅ |
| **关键要点** | JSON文本框 | 见下方格式说明 | ❌ |
| **案例配图** | 文件上传 | 支持JPG、PNG等图片格式 | ❌ |

**关键要点JSON格式**:

```json
[
  "35岁单身专业人士，年收入80万",
  "重点关注职业发展和未来家庭规划",
  "配置储蓄和重疾保障的平衡组合",
  "总保费约占年收入的12%，符合合理范围",
  "预留流动性资金应对突发需求"
]
```

#### 🕐 时间信息（只读，自动折叠）

| 字段 | 说明 |
|------|------|
| **创建时间** | 自动记录，不可修改 |
| **更新时间** | 自动更新，不可修改 |

## 4. 常见问题

### Q1: JSON格式错误怎么办？

**A**: 确保JSON格式正确：
- 使用双引号（不是单引号）
- 每个对象用逗号分隔
- 数字不用引号
- 最后一项后面不要逗号

**检查工具**: 可以使用在线JSON验证器（如 jsonlint.com）验证格式

### Q2: 年缴保费总额如何计算？

**A**: 将 `recommended_products` 数组中所有产品的 `annual_premium` 相加即可。

例如: 50000 + 30000 + 20000 = 100000

### Q3: 如何批量修改案例状态？

**A**:
1. 在列表页面勾选需要修改的案例
2. 在底部下拉菜单选择"启用选中的案例"或"禁用选中的案例"
3. 点击"执行"按钮

### Q4: 排序序号如何生效？

**A**:
- 数字越小，排序越靠前
- 建议使用10、20、30这样的间隔数字，方便后续插入新案例
- 相同排序序号时，按人生阶段和创建时间排序

### Q5: 案例配图有大小限制吗？

**A**:
- 建议图片大小不超过5MB
- 支持格式：JPG、PNG、GIF、WebP
- 建议分辨率：1200x800 或相似比例

### Q6: 如何快速复制案例？

**A**:
1. 打开要复制的案例
2. 修改标题（避免重复）
3. 调整需要修改的字段
4. 点击"保存为新对象"按钮（底部）

### Q7: 搜索功能支持模糊匹配吗？

**A**: 是的，搜索支持模糊匹配，会在以下字段中搜索：
- 案例标题
- 案例详细说明
- 家庭结构
- 保险需求

### Q8: 可以导出案例数据吗？

**A**: 目前Admin界面暂不支持直接导出，但可以：
1. 使用数据库查询导出
2. 开发自定义导出功能（需要编程）
3. 使用第三方Django扩展包（如 django-import-export）

## 5. 最佳实践

### ✅ 标题命名

**推荐格式**: "年龄+身份+保险规划"

**示例**:
- "30岁新婚夫妇保险规划"
- "40岁中年家庭全面保障方案"
- "25岁单身职场新人保险配置"

### ✅ 排序规划

**建议策略**:

```
扶幼保障期: 10, 11, 12, ...
收入成长期: 20, 21, 22, ...
责任高峰期: 30, 31, 32, ...
责任递减期: 40, 41, 42, ...
退休期:     50, 51, 52, ...
```

### ✅ 数据完整性

**必填字段检查清单**:
- [ ] 案例标题是否清晰描述目标人群
- [ ] 人生阶段是否正确选择
- [ ] 客户年龄是否合理（与人生阶段匹配）
- [ ] 年收入是否准确
- [ ] 家庭结构是否详细描述
- [ ] 保险需求是否明确
- [ ] 推荐产品JSON格式是否正确
- [ ] 年缴保费总额是否与推荐产品一致
- [ ] 案例详细说明是否充分
- [ ] 关键要点是否提炼精准

### ✅ 内容质量

**案例详细说明应包含**:
1. 客户背景分析
2. 保险需求分析
3. 推荐方案说明
4. 保费预算分析
5. 后续调整建议

**关键要点应包含**:
1. 客户基本信息（年龄、收入、家庭）
2. 保险配置特点
3. 保费与收入比例
4. 方案亮点或特色

## 7. 技术支持

### 常见错误处理

#### 错误1: "JSON格式不正确"

**原因**: JSON语法错误

**解决**:
- 检查引号（必须是双引号）
- 检查逗号位置
- 检查括号匹配
- 使用JSON验证工具

#### 错误2: "必填字段未填写"

**原因**: 遗漏必填字段

**解决**: 检查以下必填字段是否都已填写：
- 案例标题
- 人生阶段
- 客户年龄
- 年收入
- 家庭结构
- 保险需求
- 推荐产品列表
- 年缴保费总额
- 案例详细说明

#### 错误3: "文件上传失败"

**原因**: 文件大小或格式问题

**解决**:
- 检查文件大小（不超过5MB）
- 检查文件格式（JPG、PNG、GIF）
- 检查文件名（避免特殊字符）
