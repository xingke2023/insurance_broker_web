## 📁 自定义JSON文件格式

如果你想创建自己的案例数据，可以创建一个JSON文件，格式如下：

```json
[
  {
    "title": "案例标题",
    "life_stage": "扶幼保障期",
    "customer_age": 30,
    "annual_income": 600000,
    "family_structure": "家庭结构描述",
    "insurance_needs": "保险需求描述",
    "budget_suggestion": "预算建议",
    "recommended_products": [
      {
        "product_name": "产品名称",
        "company": "保险公司",
        "annual_premium": 40000,
        "coverage_type": "储蓄",
        "reason": "推荐理由"
      }
    ],
    "total_annual_premium": 73000,
    "case_description": "案例描述（150-300字）",
    "key_points": [
      "要点1",
      "要点2",
      "要点3"
    ],
    "sort_order": 10,
    "is_active": true
  }
]
```

### 字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `title` | string | 是 | 案例标题，唯一标识 |
| `life_stage` | string | 是 | 人生阶段（5个选项） |
| `customer_age` | integer | 是 | 客户年龄 |
| `annual_income` | decimal | 是 | 年收入 |
| `family_structure` | string | 否 | 家庭结构描述 |
| `insurance_needs` | string | 否 | 保险需求 |
| `budget_suggestion` | string | 否 | 预算建议 |
| `recommended_products` | JSON | 否 | 推荐产品列表 |
| `total_annual_premium` | decimal | 是 | 年缴保费总额 |
| `case_description` | text | 否 | 案例描述 |
| `key_points` | JSON | 否 | 关键要点列表 |
| `sort_order` | integer | 否 | 排序序号（默认0） |
| `is_active` | boolean | 否 | 是否启用（默认true） |

### 人生阶段选项（必须使用以下之一）
- `扶幼保障期`
- `收入成长期`
- `责任高峰期`
- `责任递减期`
- `退休期`

## 🖼️ 添加案例图片

### 图片要求
- **格式**: JPG, PNG, WebP
- **尺寸**: 建议 800x600 或 1200x900
- **大小**: 不超过2MB
- **命名**: 使用描述性名称，如 `family-30s.jpg`

### 添加方法

#### 方法1: 通过Django Admin上传
1. 登录Django Admin: `http://localhost:8017/admin/`
2. 进入"客户案例"管理页面
3. 点击要编辑的案例
4. 在"案例配图"字段上传图片
5. 点击"保存"

#### 方法2: 手动复制到media目录
```bash
# 创建目录（如果不存在）
mkdir -p /var/www/harry-insurance2/media/customer_cases/

# 复制图片文件
cp /path/to/your/image.jpg /var/www/harry-insurance2/media/customer_cases/

# 设置权限
chmod 644 /var/www/harry-insurance2/media/customer_cases/image.jpg
```

然后在Django Admin中手动设置图片路径为: `customer_cases/image.jpg`

## 🔄 重新导入数据

### 场景1: 更新现有案例
如果你修改了案例内容并想重新导入：

```bash
# 1. 先清空现有数据
python manage.py import_customer_cases --clear

# 2. 重新导入
python manage.py import_customer_cases
```

### 场景2: 只添加新案例
编辑 `api/management/commands/import_customer_cases.py` 文件，在 `get_sample_data()` 方法中添加新案例，然后运行：

```bash
python manage.py import_customer_cases
```

系统会自动跳过已存在的案例（基于title字段判断）。

## 🐛 常见问题

### Q1: 导入时提示"案例已存在"
**原因**: 数据库中已存在相同标题的案例

**解决方案**:
```bash
# 方案1: 使用 --clear 选项清空后重新导入
python manage.py import_customer_cases --clear

# 方案2: 修改案例标题，使其唯一
# 编辑 import_customer_cases.py，修改 title 字段
```

### Q2: JSON格式错误
**原因**: JSON文件格式不正确

**解决方案**:
```bash
# 验证JSON格式
python3 -m json.tool your_cases.json

# 如果有错误会显示具体位置
```

### Q3: 人生阶段无效
**原因**: life_stage字段值不在允许的5个选项中

**解决方案**:
确保使用以下之一：
- 扶幼保障期
- 收入成长期
- 责任高峰期
- 责任递减期
- 退休期

### Q4: 导入后前端不显示
**原因**: 可能是API服务未重启或前端缓存

**解决方案**:
```bash
# 1. 重启Django服务
sudo supervisorctl restart harry-insurance:harry-insurance-django

# 2. 清除浏览器缓存
# 按 Ctrl+Shift+R 强制刷新页面

# 3. 检查API响应
curl "http://localhost:8017/api/customer-cases/"
```
