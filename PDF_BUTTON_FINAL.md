# 产品PDF识别按钮 - 最终版本

## ✅ 功能位置

在产品编辑页面的「计划书内容」区域：

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
计划书内容
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

📄 选择计划书PDF文件
   [选择文件] 未选择任何文件

   [🔍 仅识别并保存]  ← 新增按钮（橙色）

💡 提示：
• 保存（页面底部）: 完整更新 Base64编码 + 计划书详情
• 仅识别并保存（上方按钮）: 只更新计划书详情，不更新Base64

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

## 🎯 两种处理方式

### 方式1: 完整保存（页面底部"保存"按钮）

**操作步骤**：
```
1. 选择PDF文件
2. 滚动到页面底部
3. 点击"保存"按钮
```

**处理内容**：
```
✅ 更新 plan_pdf_base64（PDF的Base64编码）
✅ OCR识别
✅ 更新 plan_details（识别结果）
```

**适用场景**：
- 首次上传产品PDF
- PDF文件内容有更新
- 需要前端能下载最新PDF

### 方式2: 仅识别并保存（PDF文件选择器下方按钮）

**操作步骤**：
```
1. 选择PDF文件
2. 直接点击"🔍 仅识别并保存"按钮（就在文件选择器下方）
```

**处理内容**：
```
⏭️ 不更新 plan_pdf_base64（保持原值）
✅ OCR识别
✅ 更新 plan_details（识别结果）
```

**适用场景**：
- PDF文件未变化，只想重新OCR
- OCR识别质量不佳，想重新识别
- 测试不同的识别效果
- 节省时间和数据库空间

## 📊 对比表

| 操作 | 完整保存 | 🔍 仅识别并保存 |
|------|---------|---------------|
| **按钮位置** | 页面底部 | PDF文件选择器下方 |
| **按钮颜色** | 蓝色（默认） | 橙色 |
| **更新Base64** | ✅ 是 | ❌ 否 |
| **OCR识别** | ✅ 是 | ✅ 是 |
| **更新详情** | ✅ 是 | ✅ 是 |
| **处理时间** | 较长（10-20秒） | 较短（5-15秒） |
| **点击次数** | 需要滚动 | 就在附近 |

```
4. 等待重新识别

结果：
⏭️ Base64保持不变（不重复保存）
✅ 计划书详情已重新识别
```

### 工作流3: 测试OCR效果

```

1. 选择PDF文件

5. 重复直到满意

优势：
⏭️ 不重复保存Base64（节省空间）
✅ 快速重试（无需重新上传）
```

## ⚠️ 重要提示

### 1. 按钮位置很关键

```
方便程度：
🔍 仅识别并保存（文件选择器下方）>>> 保存（页面底部）

原因：
- 选择文件后立即可见
- 无需滚动到底部
- 操作更快捷
```

### 2. 两个按钮都会覆盖内容

无论使用哪个按钮，都会**覆盖** `plan_details` 字段！

- 如果已手动编辑，请先备份
- 如果只想保留手动内容，不要点击任何按钮

### 3. 必须选择文件

两个按钮都需要先选择PDF文件才能使用。
