# PDF页脚擦除工具 - 界面优化更新

## 更新内容

### 1. 标题文字优化

**修改前**：页面范围选择（只保存指定页面）
**修改后**：只保存指定页面

更简洁直观，用户一眼就能明白这个功能的作用。

### 2. 取消页面范围自动记忆

**修改原因**：
- 页面范围通常是临时操作，不需要每次都记住
- 避免用户忘记之前的设置，导致意外裁剪页面
- 每次处理新文件时，默认保存全部页面更安全

**默认值**：
- `saveStartPage`: 1（从第1页开始保存）
- `saveEndPage`: 0（保存到最后一页，即全部保存）

### 3. 保留自动记忆的功能

以下功能仍然会自动保存到 LocalStorage：

✅ **处理开始页码** (`process_start_page`)
✅ **处理结束页码** (`process_end_page`)
✅ **起始页码编号** (`page_number_start`)
✅ **自定义页脚文字** (`custom_text`)
✅ **擦除区域设置** (`remove_areas`)

❌ **不再自动保存**：
- `save_start_page`（保存起始页）
- `save_end_page`（保存结束页）

```
用户上传10页PDF
上次设置了：save_start_page=3, save_end_page=8
本次上传新文件时，自动应用上次设置

```

### 修改后

```
用户上传10页PDF

结果：默认保存全部页面，用户需要主动设置才会裁剪
```

## 技术实现

### 状态初始化

**修改前**（从 LocalStorage 读取）：
```javascript
const [saveStartPage, setSaveStartPage] = useState(() => {
  const saved = localStorage.getItem('pdf_save_start_page');
  return saved ? parseInt(saved) : 1;
});

const [saveEndPage, setSaveEndPage] = useState(() => {
  const saved = localStorage.getItem('pdf_save_end_page');
  return saved ? parseInt(saved) : 0;
});
```

**修改后**（固定默认值）：
```javascript
const [saveStartPage, setSaveStartPage] = useState(1);
const [saveEndPage, setSaveEndPage] = useState(0);
```

### 移除 useEffect 保存

**删除了以下代码**：
```javascript
useEffect(() => {
  localStorage.setItem('pdf_save_start_page', saveStartPage.toString());
}, [saveStartPage]);

useEffect(() => {
  localStorage.setItem('pdf_save_end_page', saveEndPage.toString());
}, [saveEndPage]);
```

## 清理旧数据（可选）

如果需要清理用户浏览器中的旧设置，可以在控制台运行：

```javascript
// 清理旧的页面范围设置
localStorage.removeItem('pdf_save_start_page');
localStorage.removeItem('pdf_save_end_page');
```

或者在组件中添加一次性清理（可选）：
```javascript
useEffect(() => {
  // 清理旧版本的LocalStorage数据
  localStorage.removeItem('pdf_save_start_page');
  localStorage.removeItem('pdf_save_end_page');
}, []); // 只运行一次
```

## 用户操作流程

### 场景1：完整处理所有页面

```
1. 上传10页PDF
2. 设置擦除区域和页码参数
3. 不修改"只保存指定页面"（默认1到0）
4. 点击"移除页脚"
5. 结果：10页完整PDF，全部处理
```

### 场景2：只保留部分页面

```
1. 上传20页PDF
2. 设置擦除区域和页码参数
3. 修改"只保存指定页面"：
   - 保存起始页：5
   - 保存结束页：15
4. 点击"移除页脚"
5. 结果：11页PDF（第5-15页），第1-4页和16-20页丢弃
```

### 场景3：下次处理新文件

```
1. 上传新的15页PDF
2. 擦除区域和页码参数自动恢复上次设置（便利）
3. "只保存指定页面"重置为默认（1到0，全部保存）（安全）
4. 用户需要主动设置才会裁剪页面
```

## 注意事项

1. **默认行为变化**：现在每次上传新文件，默认保存全部页面
2. **主动操作**：需要裁剪时，用户必须手动设置范围
3. **重置按钮**：点击"重置"会将所有参数恢复为默认值（包括裁剪范围）
4. **不影响已保存的其他参数**：擦除区域、页码设置等依然会自动记忆

## 总结

此次更新让"只保存指定页面"功能更加：

✅ **安全**：默认保存全部，避免意外丢失页面
✅ **直观**：标题简化，一目了然
✅ **合理**：临时操作不需要记忆，常用设置保留记忆
✅ **友好**：用户不会因为忘记上次设置而困惑

**适用原则**：
- 常用的、稳定的参数 → 自动记忆（擦除区域、页码设置）
- 临时的、可变的参数 → 每次重置（裁剪范围）
