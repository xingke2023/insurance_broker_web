# 产品PDF更新选项说明

## ✅ 功能已更新

在Admin后台编辑产品时，现在可以灵活控制PDF上传的行为。

## 📋 上传选项

### 1. 上传计划书PDF
- **字段**: `plan_pdf`
- **说明**: 选择要上传的PDF文件
- **限制**: 最大20MB

### 2. 覆盖计划书详情
- **字段**: `overwrite_plan_details`
- **类型**: 复选框
- **默认**: 勾选（启用）

## 🎯 使用场景

### 场景1：首次上传PDF（勾选"覆盖计划书详情"）

```
✅ 勾选"覆盖计划书详情"
↓
上传PDF文件
↓
系统执行：
1. PDF → Base64编码 → 保存到 plan_pdf_base64
2. PDF → OCR识别 → 保存到 plan_details
↓
结果：
✅ PDF已转换为Base64编码（XXX字符）
✅ PDF OCR解析成功！已将内容保存到"计划书详情"字段（XXX字符）
```

### 场景2：只更新PDF文件（不勾选"覆盖计划书详情"）

```
❌ 取消勾选"覆盖计划书详情"
↓
上传新的PDF文件
↓
系统执行：
1. PDF → Base64编码 → 保存到 plan_pdf_base64
2. 跳过OCR识别，保留原有的 plan_details 内容
↓
结果：
✅ PDF已转换为Base64编码（XXX字符）
ℹ️ 已更新PDF Base64编码，保留原有"计划书详情"内容
```

## 💡 最佳实践

### 何时勾选"覆盖计划书详情"

✅ **应该勾选**：
- 首次上传产品计划书PDF
- PDF内容有重大更新
- 希望重新OCR识别文档内容
- plan_details字段为空或内容过时

❌ **不应勾选**：
- 仅更新PDF文件版本（内容未变）
- 已手动编辑过plan_details，不想被覆盖
- 只需要更新前端下载的PDF文件
- OCR识别质量不好，想保留人工编辑的内容

## 📊 字段对比

| 操作 | plan_pdf_base64 | plan_details |
|------|----------------|--------------|
| 勾选✅ + 上传PDF | ✅ 更新 | ✅ OCR覆盖 |
| 不勾选❌ + 上传PDF | ✅ 更新 | ⏭️ 保留原值 |
| 不上传PDF | ⏭️ 保持不变 | ⏭️ 保持不变 |

## 🔍 实际案例

### 案例1：新产品录入

```
步骤：
1. 创建新产品记录
2. 勾选✅"覆盖计划书详情"
3. 上传计划书PDF
4. 保存

结果：
- plan_pdf_base64: 完整的PDF Base64编码
- plan_details: OCR识别的计划书文本内容
```

### 案例2：更新PDF版本

```
场景：保险公司发布了新版本的计划书PDF，但内容99%相同，
只是更新了公司logo或版本号

步骤：
1. 编辑现有产品记录
2. 取消勾选❌"覆盖计划书详情"
3. 上传新版本PDF
4. 保存

结果：
- plan_pdf_base64: 新PDF的Base64编码（前端可下载新版本）
- plan_details: 保留原有内容（无需重新OCR）
```

### 案例3：手动修正OCR错误

```
场景：OCR识别有错误，已手动修正plan_details内容，
现在需要更新PDF文件

步骤：
1. 编辑产品记录
2. 取消勾选❌"覆盖计划书详情"
3. 上传正确的PDF
4. 保存

结果：
- plan_pdf_base64: 正确的PDF编码
- plan_details: 保留手动修正的内容（不被覆盖）
```

## 🛡️ 错误处理

### OCR识别失败

即使OCR失败，Base64编码也会保存：

```
✅ PDF已转换为Base64编码（XXX字符）
⚠️ PDF OCR解析失败: 错误信息（Base64已保存）
```

你仍然可以：
- 在前端下载PDF文件
- 手动编辑plan_details字段

### 文件上传失败

- 文件类型错误：只接受PDF格式
- 文件过大：最大20MB
- 网络中断：需要重新上传

## ✅ 总结

新增的「覆盖计划书详情」选项让您可以：

1. ✅ **灵活更新**：可以只更新PDF，不影响已编辑的文本内容
2. ✅ **节省时间**：不需要每次都等待OCR识别
3. ✅ **保护数据**：避免意外覆盖手动修正的内容
4. ✅ **提升体验**：根据实际需求选择合适的更新方式

**默认行为保持不变**：勾选状态下，上传PDF会自动OCR识别并覆盖plan_details，适合首次上传的场景。
