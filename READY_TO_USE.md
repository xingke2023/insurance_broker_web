## 📋 功能概述

当用户上传PDF时，系统会**自动触发两个并行任务**：

1. **原有OCR流程**（保留）
   - OCR识别 → 提取表格 → 分析概要
   - 保存到：content, tablecontent, tablesummary等字段

2. **新Gemini直接提取**（新增）
   - Gemini直接分析PDF → 智能选择最佳表格 → 提取完整数据
   - 保存到：**table1字段**（JSON格式）

## 🔧 技术细节

### Gemini提示词

```
分析table里包含列名为"保单年度终结"或"保單年度終結"的table
可能会有好几个这样的表，有些表可能会跨页但其实是属于一个表
选取一个表，这个表必须满足下面条件：
1.不是悲观或乐观（非不同投资回报），也不是身故，
2.如果有提取就选提取表（字段一般带提取两个字），如果没有就选退保价值表（或者现金价值而不是身故）
3.尽量选取跨页总行数较多的表，最好不是抽样年份展示的表
返回这个表所有行包括跨页行的数据的json格式（第一行是字段名，去除空格换行的数组格式，减少token浪费）
```

### 配置参数

- **模型**：gemini-3-flash-preview
- **thinkingBudget**：1024（最小思考预算）
- **media_resolution**：MEDIA_RESOLUTION_LOW（低分辨率，加速）
- **response_mime_type**：application/json（纯JSON输出）
- **重试次数**：2次（间隔60秒）

## 📝 注意事项

1. **数据类型**：当前所有数据都是字符串格式（如"1,000,000"）
2. **并行执行**：两个任务同时运行，互不影响
3. **向后兼容**：原有OCR流程完全保留
4. **错误处理**：自动重试2次，失败不影响OCR流程
