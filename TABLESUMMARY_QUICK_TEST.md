# 表格概要快速测试工具

## 功能说明

这个脚本可以快速从OCR文本文件中提取保险计划书的表格概要信息。

**核心功能**：
- 读取OCR识别后的计划书文本文件
- 调用DeepSeek API分析所有包含"保单年度终结"的表格
- 提取表格名称、行数、基本字段
- 自动保存分析结果到文件

## 使用方法

### 1. 基本用法

```bash
python3 test_extract_tablesummary.py <OCR文本文件路径>
```

### 2. 使用示例

```bash
# 示例1：使用相对路径
python3 test_extract_tablesummary.py "中銀集團人壽 保險有限公司.txt"

# 示例2：使用绝对路径
python3 test_extract_tablesummary.py /var/www/harry-insurance2/ocr_files/plan.txt

# 示例3：处理其他保险公司
python3 test_extract_tablesummary.py "友邦保险计划书_ocr.txt"
```

### 3. 输出结果

脚本会：
1. **屏幕输出**：显示完整的表格概要分析结果
2. **文件保存**：自动保存到 `<原文件名>_tablesummary.txt`

## 工作原理

### 分析流程

```
读取OCR文本文件
    ↓
截取前120,000字符（API限制）
    ↓
构建Prompt（与Celery任务完全一致）
    ↓
调用DeepSeek API分析
    ↓
提取表格概要（名称、行数、字段）
    ↓
清理markdown标记
    ↓
输出并保存结果
```

### 使用的Prompt

```
以保单年度终结为坐标，分析以下保险计划书中的所有表格。

要求：
1. 识别所有以"保单年度终结"为坐标的表格
2. 有些表格可能跨度好几个页面，但只算一张表，请完整识别
3. 对每个表格提取：表详细名称、行数、基本字段

只输出结果，不要有任何解释说明。
```

### API配置

- **模型**：`deepseek-chat`
- **Temperature**：0.3（保持一致性）
- **Max Tokens**：2000
- **内容限制**：前120,000字符

## 适用场景

### ✅ 适合使用

1. **快速验证OCR质量**：检查OCR是否正确识别表格结构
2. **调试表格提取**：测试不同保险公司的表格格式
3. **Prompt优化**：调整Prompt后快速验证效果
4. **离线分析**：不依赖数据库，独立运行

1. **批量处理**：需要处理多个文件时，应使用Celery任务链
2. **完整数据提取**：只提取概要，不提取具体数值
3. **生产环境**：生产环境应使用完整的Celery处理流程

## 常见问题

### Q1: 提示"DEEPSEEK_API_KEY环境变量未设置"

**解决方法**：
```bash
# 检查.env文件
cat .env | grep DEEPSEEK_API_KEY

# 如果没有，添加：
echo "DEEPSEEK_API_KEY=sk-your-key-here" >> .env
```

### Q2: 文件路径包含空格怎么办？

**解决方法**：使用引号包裹文件名
```bash
python3 test_extract_tablesummary.py "中銀集團人壽 保險有限公司.txt"
```

### Q3: 返回结果为空或不准确？

**可能原因**：
1. OCR内容质量差，表格结构识别错误
2. 文档中没有"保单年度终结"这个关键词
3. 表格格式特殊，与常规格式差异较大

**调试方法**：
1. 检查OCR文本文件，确认是否有清晰的表格结构
2. 查看是否有"保单年度终结"、"保單年度終結"等关键词
3. 尝试调整Prompt（修改脚本第28-59行）

### Q4: API调用超时或失败？

**解决方法**：
1. 检查网络连接
2. 验证API密钥是否有效
3. 查看DeepSeek API配额是否用完
