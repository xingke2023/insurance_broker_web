# 表格查看功能使用指南

## 功能位置

"查看表格"按钮位于**文档详情页面**中的**"计划书表格列表"卡片**内。

## 访问路径

1. **方式1：从文档管理页面**
   ```
   Dashboard → 计划书分步骤分析 → 点击任意文档 → 查看文档详情页
   ```

2. **方式2：直接访问**
   ```
   http://localhost:8008/document/<document_id>
   ```

## 表格详情页

点击"查看表格"按钮后，会在新标签页打开表格详情页面。

### 页面结构

**1. 顶部导航栏**
- ← 返回按钮
- 📊 表格详情标题
- ⬇ 下载HTML按钮

**2. 表格信息卡片**
显示表格的元数据：
- 所属文档：xxx.pdf
- 表格编号：表格 1
- 表格名称：詳細說明 - 退保價值
- 总行数：101 行
- 创建时间：2024-01-19 10:30:00
- 字段列表：保单年度终结,缴付保费总额,退保价值

**3. 表格内容卡片**
- **渲染预览**：直接渲染HTML表格，可以查看表格样式
- **HTML源代码**：折叠区域，点击"查看HTML源代码"展开

### 功能特性

✅ **实时渲染**
- 表格HTML会被直接渲染显示
- 自动应用美化样式（边框、悬停效果等）
- 支持横向滚动（超宽表格）

✅ **下载功能**
- 点击右上角"下载HTML"按钮
- 自动下载格式：`文档名_table编号.html`
- 可以在本地浏览器打开查看

✅ **源码查看**
- 点击"查看HTML源代码"展开
- 显示完整的HTML标签内容
- 方便技术人员调试和分析

## 使用场景

### 场景1：查看特定表格数据
1. 打开文档详情页
2. 滚动到"计划书表格列表"卡片
3. 找到需要的表格（如"退保價值表"）
4. 点击"查看表格"按钮
5. 在新标签页查看详细表格内容

### 场景2：导出表格数据
1. 打开表格详情页
2. 点击右上角"下载HTML"按钮
3. 保存HTML文件到本地
4. 用浏览器打开查看或进一步处理

### 场景3：对比多个表格
1. 在文档详情页打开多个表格（每个都在新标签页）
2. 在不同标签页之间切换对比数据
3. 可同时查看"退保价值"和"身故赔偿"等

### 场景4：技术分析
1. 打开表格详情页
2. 展开"查看HTML源代码"
3. 分析表格结构和数据格式
4. 用于开发数据解析功能

## 技术细节

### 前端实现

**DocumentDetail.jsx 修改**
- 新增 `isPlanTablesOpen` 状态管理折叠
- 新增"计划书表格列表"卡片渲染
- 使用 `window.open()` 在新标签页打开表格

**TableDetail.jsx 组件**
- 独立页面组件
- 调用 `GET /api/ocr/tables/{table_id}/` API
- 使用 `dangerouslySetInnerHTML` 渲染HTML
- 支持下载HTML文件

**路由配置**
```javascript
<Route path="/table/:tableId" element={
  <ProtectedRoute>
    <TableDetail />
  </ProtectedRoute>
} />
```

### API接口

**1. 获取文档详情（包含表格列表）**
```
GET /api/ocr/documents/{document_id}/
```

响应新增 `plan_tables` 字段：
```json
{
  "status": "success",
  "data": {
    "plan_tables": [
      {
        "id": 1,
        "table_number": 1,
        "table_name": "詳細說明 - 退保價值",
        "row_count": 101,
        "fields": "保单年度终结,缴付保费总额...",
        "created_at": "2024-01-19T..."
      }
    ]
  }
}
```

**2. 获取单个表格详情**
```
GET /api/ocr/tables/{table_id}/
```

响应包含完整HTML：
```json
{
  "status": "success",
  "data": {
    "id": 1,
    "table_number": 1,
    "table_name": "詳細說明 - 退保價值",
    "row_count": 101,
    "fields": "保单年度终结,缴付保费总额...",
    "html_source": "<table>\n  <tr>...",
    "plan_document": {
      "id": 123,
      "file_name": "xxx.pdf"
    }
  }
}
```

## 数据来源

表格数据来自Celery自动任务处理流程：

1. **用户上传PDF** → OCR识别
2. **步骤3：提取表格概要**
   - 调用DeepSeek AI分析表格结构
   - ⭐ **同时提取各个表格的HTML源代码**
   - 保存到 `PlanTable` 数据库表
3. 前端通过API获取并展示

## 常见问题

### Q1: 为什么有些文档没有"计划书表格列表"卡片？

**可能原因：**
- 文档处理尚未完成步骤3
- OCR内容中没有包含"保单年度终结"的表格
- 文档是在功能上线前上传的（需要重新处理）

**解决方案：**
- 等待Celery任务完成（查看处理状态）
- 重新上传文档

### Q2: 表格显示不完整或格式错误？

**可能原因：**
- OCR识别质量问题
- 跨页表格合并不完整
- HTML标签缺失

**解决方案：**
- 下载HTML文件本地查看
- 在Admin后台查看原始HTML
- 联系技术支持

### Q3: 如何一次性查看所有表格？

**当前方案：**
- 依次点击每个表格的"查看表格"按钮
- 每个表格在新标签页打开
- 在浏览器标签页之间切换查看

**未来优化：**
- 可开发"全部表格预览"功能
- 在一个页面中展示所有表格

### Q4: 表格数据可以导出为Excel吗？

**当前状态：**
- 仅支持下载HTML格式
- HTML可在Excel中打开（导入网页数据）

**未来功能：**
- 计划支持导出CSV/Excel格式
- 添加数据解析和格式化功能
